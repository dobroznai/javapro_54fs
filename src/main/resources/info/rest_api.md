### GET - получает данные (например, вернуть список пользователей)

GET/api/users - безопасный
идемпатентность - это когда метод можно множество раз использовать не меняя данные (какое количество в единицу времени)

### POST - создает новый обьект (к примеру создание нового пользователя в системе)

POST/api/user - не безопасен
не иденпатентный

### PUT - обновляет обьект (происходит обновление обьекта, замена обьекта по факту)

PUT/api/user - не безопасен
иденпотентен

### PATCH - частичное обновление обьекта, не нужно отправлять весь обьект, не все арi поддерживают

иденпотентный: да и нет

### DELETE - удаление обьекта (по идентификатору)

дейстыие необратимое
иденпотентный

#### 200 - ОК все успешно

#### 201 - CREATED обьект успешно создан POST

#### 400 - BAD REQUEST - ошибка данных

#### 404 - NOT FOUND - обьект не найден

#### 500 - INTERNAL SERVER ERROR - ошибка сервера
